# 单只股票牛熊状态判断体系：设计文档 (v2.1)

> 对应实现文件：`src/market/stock_state_detector.py`、`web/services/market_regime_service.py`

---

## 一、设计目标

对**单只股票**（A股/港股）的当前市场状态进行量化评判，输出"牛市/熊市/中性"状态及置信度分数。

核心设计原则：
- **自适应**：指标基于个股自身历史数据，而非市场统一阈值，使蓝筹、成长股、题材股可用同一套框架
- **正交性**：每个维度捕捉一类独立信息，避免多维度重复计量同一现象导致权重扭曲
- **可解释性**：每个维度有清晰的市场意义，便于在信号出现时理解原因
- **A股适配**：充分考虑A股散户主导、涨跌停机制、题材驱动等市场特性

---

## 二、评分体系总览

总分100分，阈值：**≥70分=牛市，≤40分=熊市，中间=中性**

| 维度 | 分值范围 | 独立信息 | 核心指标 |
|------|---------|---------|---------|
| 趋势分 | -30 ~ +30 | 价格运行方向 | EMA短/中/长排列 + SMA方向锚 |
| 动能分 | -25 ~ +25 | 趋势力量变化 | 两期ROC之差（加速度）× 动态ATR阈值 |
| 量价分 | 0 ~ +20 | 资金真实参与意愿 | 20日涨跌日量比 + 量价背离检测 |
| 换手率分 | 0 ~ +15 | 市场参与热度 | N日换手率 / 历史均值 |
| 波动分 | -10 ~ +10 | 波动状态 × 趋势含义 | ATR百分位（40日历史）+ 趋势联动 |

## 四、各维度设计详解

### 4.1 趋势分（±30分）

**核心问题**：价格运行的整体方向是什么？

**为什么用EMA而非SMA**：
A股散户广泛使用MACD（内部全部基于EMA），EMA均线是市场的"自我实现预言"——大量散户在EMA支撑位买入，EMA压力位卖出，使EMA均线真实地成为支撑/压力位。此外，A股涨跌停板制度导致价格变化集中且剧烈，EMA对近期数据赋予更高权重，能更快响应。

**为什么加入SMA方向锚**：
短期EMA排列可以被游资通过短期拉升制造假信号。加入更长周期的简单均线作为中期方向验证：
- 完美多头排列 + 价格在SMA锚上方：真正的上升趋势，+30分
- 完美多头排列 + 价格在SMA锚下方：可能只是熊市中的反弹，+15分（降半）

**评分规则**：

| EMA排列 | 价格vs锚 | 得分 |
|---------|---------|------|
| EMA短 > EMA中 > EMA长 | 在锚上方 | +30 |
| EMA短 > EMA中 > EMA长 | 在锚下方 | +15 |
| EMA短 > EMA中 | 在锚上方 | +10 |
| EMA短 > EMA中 | 在锚下方 | +5 |
| EMA短 < EMA中 < EMA长 | 在锚下方 | -30 |
| EMA短 < EMA中 < EMA长 | 在锚上方 | -15 |
| EMA短 < EMA中 | 在锚下方 | -10 |
| EMA短 < EMA中 | 在锚上方 | -5 |
| 混乱排列 | - | 0 |

---

### 4.2 动能分（±25分）

**核心问题**：上涨/下跌的力量在增强还是减弱？

**为什么用加速度而非ROC绝对值**：
原始ROC（价格变化率）与趋势分（均线排列）高度重叠——均线多头排列时ROC基本为正；均线空头排列时ROC基本为负。两者都在说"价格比过去高/低"。

**加速度 = 当期ROC - 前期ROC**，测量的是趋势力量是否在增强，与趋势方向无关，真正正交：
- 股票持续上涨但速度越来越慢（加速度转负）：见顶前的早期预警
- 股票在下跌但跌幅越来越小（加速度转正）：底部反转的先行信号
- 股票大涨后加速继续：主升浪爆发确认

**计算方法**：
```
当期ROC = (close[-1] - close[-N-1]) / close[-N-1]   // 最近N天变化率
前期ROC = (close[-N-1] - close[-2N-1]) / close[-2N-1] // 前N天变化率
加速度 = 当期ROC - 前期ROC
动态阈值 = 1.5 × ATR%（自适应不同波动率股票）
```

**为什么需要四象限评分（而非纯加速度评分）**：

纯加速度评分存在一个根本问题：前期涨幅远大于当期涨幅时，加速度为负（急剧减速），会给出极低分——即使股票当前仍在上涨。

典型案例：股票在前一个月暴涨+23%，当前月上涨+11%，加速度 = -12%（急剧减速），纯加速度评分给 -25 分，导致"上涨中的股票被判断为熊市"。

**正确逻辑**：加速度是对趋势方向的修正，不能脱离方向单独使用：
- 价格在涨（current_roc > 0）→ 基准是正面的，加速度只做调节（从+25到-5）
- 价格在跌（current_roc < 0）→ 基准是负面的，加速度只做调节（从+5到-25）

**四象限评分表**：

| 当期ROC方向 | 加速度 | 得分 | 含义 |
|------------|--------|------|------|
| 上涨（>0） | > 动态阈值 | +25 | 涨势加速，主升浪爆发 |
| 上涨（>0） | 0 ~ 动态阈值 | +15 | 涨势平稳，趋势健康 |
| 上涨（>0） | -动态阈值 ~ 0 | +10 | 涨势轻微减速，仍为正面 |
| 上涨（>0） | < -动态阈值 | -5 | 涨势急速衰减，见顶预警 |
| 下跌（<0） | > 动态阈值 | +5 | 跌势快速减弱，底部信号 |
| 下跌（<0） | 0 ~ 动态阈值 | -5 | 跌势缓慢减弱，仍偏弱 |
| 下跌（<0） | -动态阈值 ~ 0 | -15 | 跌势轻微加速，偏负面 |
| 下跌（<0） | < -动态阈值 | -25 | 跌势急速加速，深度下跌 |

---

### 4.3 量价分（0~+20分）

**核心问题**：价格变化是否有真实的资金意愿支撑？

**核心逻辑**：
A股经典出货模式是"高价位低换手率拉升"——价格在涨但参与的人极少。通过比较上涨日和下跌日的平均成交量来识别：
- 上涨日成交量 > 下跌日成交量：真实需求，市场参与者愿意用高价买入
- 下跌日成交量 > 上涨日成交量：上涨日没有真实买盘，更多人在下跌时参与

**为什么从5日延长到20日**：5天数据极易被游资短期制造假量，20天（一个月）的统计样本更可靠。

**量价背离检测**（嵌入量价分）：
价格创近期新高但成交量萎缩，是最经典的技术顶部信号，内嵌为惩罚项：
- 5日新高 + 近期量能 < 历史均值×0.8：触发背离惩罚，基础分-5

**量比评分基础**：

| 涨跌日量比 | 基础分 |
|-----------|--------|
| > 2.0 | 18 |
| > 1.5 | 14 |
| > 1.0 | 10 |
| > 0.8 | 5 |
| ≤ 0.8 | 0 |

OBV加成：3日价格新高且无背离，+2分（最高合计20分）

---

### 4.4 换手率分（0~+15分，仅正分）

**核心问题**：有多少人在关注和参与这只股票的交易？

**A股散户情绪的特殊性**：
A股散户交易量占比超70%，情绪驱动显著。换手率（成交额/流通市值）是捕捉散户参与热度最直接的指标。数据字段：`bars_a_1d.turnover_rate`。

**为什么用比值而非绝对值**：
蓝筹银行股日换手率0.3%和题材股日换手率8%都可能是各自的"正常"水平，直接比较没有意义。用近期均值与历史均值之比，天然适配不同类型股票。

**为什么只有正分**：
换手率高本身不是坏信号，"天量是顶部"的负面含义（价格推不动了）应由动能分（加速度）来捕捉。各司其职，不重叠：
- 换手率爆量 + 价格加速度仍正 → 换手率+8，动能+25，整体看多
- 换手率爆量 + 价格加速度转负 → 换手率+8，动能-10，总分下降，发出警告

| 换手率比值（近期/历史） | 得分 |
|----------------------|------|
| 1.5 ~ 3.0 | +15（健康放量，最佳区间）|
| > 3.0 | +8（爆量，仍正面）|
| 1.0 ~ 1.5 | +10（适度放量）|
| 0.5 ~ 1.0 | +5（正常换手）|
| 0.3 ~ 0.5 | +3（缩量等待）|
| < 0.3 | 0（极度缩量）|

---

### 4.5 波动分（±10分，与趋势方向联动）

**核心问题**：当前的波动率环境对趋势判断意味着什么？

**为什么不能固定"低波动=好，高波动=坏"**：
A股历史规律：
- 牛市末期：往往低波动（温水煮青蛙，散户已经麻木）
- 熊市底部：往往高波动（千股跌停、恐慌抛售）

固定"高波动-10分"会在熊市底部给出极低分数，造成"越是底部越悲观"的反向误判。

**联动逻辑**：

| 趋势方向 | ATR百分位 | 得分 | 含义 |
|---------|---------|------|------|
| 上升趋势 | < 40% | +10 | 缩量整理蓄势 |
| 上升趋势 | 40~70% | +5 | 正常波动 |
| 上升趋势 | > 70% | -5 | 过热，注意变盘 |
| 下降趋势 | > 80% | +5 | 恐慌抛售，可能见底 |
| 下降趋势 | 40~80% | -5 | 阴跌，趋势未止 |
| 下降趋势 | < 40% | -10 | 低波动阴跌，最危险 |
| 趋势不明 | 任意 | 0 | - |

## 五、为什么删除"位置分"

v1.0 曾有"价格在20日高低区间的位置"评分（20分）：

**问题**：20日区间位置与EMA5/10/20排列高度重叠——EMA多头排列时，价格**必然**处于近期区间的上方。两个维度实际上都在描述"价格处于近期高位"这同一件事，以50%权重重复计量同一信息。

**后果**：短期暴拉的假突破股（游资短期拉升后出货），因MA和位置双高得分，被误判为牛市。

---

## 六、蓝筹 vs 成长股的适配

不需要硬编码两套不同逻辑，通过**周期参数选择**自然适配：
- **大市值蓝筹**：调用时使用长周期参数（EMA10/20/40 + SMA120锚），匹配其慢节奏
- **小市值成长股**：使用中周期参数（EMA5/10/20 + SMA60锚），匹配其快节奏

换手率分和波动分已是基于个股自身历史比较，天然适配不同类型：
- 蓝筹换手率低，但其历史基准也低，比值仍有意义
- 波动分用ATR历史百分位，也是个股自身历史对比

| 牛市 | min(1.0, (总分 - 70) / 30) |
| 熊市 | min(1.0, (40 - 总分) / 40) |

## 八、局限性与已知缺陷

1. **政策市风险未覆盖**：A股重大政策事件可以瞬间改变市场情绪，本体系无法预判，只能在价格反应后更新状态

2. **题材股的特殊性**：连续涨停板状态下技术指标失去参考价值；题材退潮时机（监管介入、炒作结束）难以用技术指标预判

3. **停牌股**：长期停牌后复牌的股票，历史均线和ATR数据被停牌前数据稀释，指标失真

4. **评分滞后性**：均线类指标天然有滞后，主升浪初期可能漏掉一部分收益。动能加速度在一定程度上弥补这一点

5. **阈值未经回测验证**：70/40阈值基于经验设定，未经系统性历史回测校准

---

## 九、未来增强方向

1. **融资净买入**（近期可行）：接入融资融券每日数据，散户加杠杆速度是A股最重要的情绪超前指标

2. **龙虎榜联动**（近期可行）：系统已有龙虎榜功能，可加入"上龙虎榜且机构净买入"为量价分加分项

3. **历史阈值自适应校准**（中期）：对每只股票基于历史回测校准最优牛/熊阈值，而非统一的70/40

4. **多周期信号共振**（中期）：短/中/长三周期同时计算，三周期方向一致时给出高置信度判断

5. **基本面过滤层**（长期）：中长线投资场景下加入ROE、净利润增速等简单基本面过滤，避免把技术上的牛市判断用在基本面持续恶化的公司

## 十、港股适配指南

| 特征 | A股 | 港股 | 适配建议 |
|------|-----|------|---------|
| 涨跌幅限制 | ±10% | 无 | 使用中/长周期参数，减少噪音 |
| 投资者结构 | 散户主导(>70%) | 机构主导 | 技术信号自我实现效果更弱 |
| 流动性分化 | 相对均匀 | 极度分化 | 日均成交额<500万港元的股票禁用换手率维度 |
| 波动分阈值 | ATR>80%=-10分 | 同等百分位波动更大 | 建议高阈值放宽至90% |
| 做空数据 | 无公开每日数据 | 每日公开 | 未来可接入做空比率维度 |
| 南下资金 | N/A | 重要边际力量 | AH两地上市股可接入南下资金净流入 |
| 牛市阈值 | ≥70分 | 建议≥75分 | 无涨跌停保护，假信号代价更高 |

## 十一、使用示例

```python
from src.market import StockStateDetector, get_cycle_config

# 使用默认中周期（成长股/波段）
detector = StockStateDetector()

# 短线/题材股
detector = StockStateDetector(cycle='short')

# 长线/蓝筹股
detector = StockStateDetector(cycle='long')

# 自定义配置（如调整港股阈值）
custom_config = get_cycle_config('medium')
custom_config['bull_threshold'] = 75  # 港股提高阈值
detector = StockStateDetector(config=custom_config)

# 检测市场状态（backtrader data需含 close/high/low/volume，可选 turnover_rate）
state = detector.detect_state(data, stock_code="000001")
print(f"状态: {state.regime.value}  置信度: {state.confidence:.2f}")
print(f"趋势:{state.ma_trend}  动能:{state.momentum}  量价:{state.volume_confirm}  换手:{state.turnover_score}  波动:{state.volatility}")
```
